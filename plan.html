4 · План по неделям (calendar 6 недель)

Week 1: Kickoff + Backend skeleton

  • ✅ Финализировать имя бота / домен (zavisi от ответов на §8 в spec)
  • ✅ Развернуть infra: Postgres + Redis + S3 + workers на Hetzner/DO
  • ✅ Auth + user-model + паспорт schema
  • ✅ API skeleton + webhook bot endpoint
  • ✅ Sentry + базовый telemetry
  • Acceptance: API работает, может зарегистрировать пользователя, может сохранить пустой паспорт.

Week 2: Онбординг + OCR

  • ✅ TG-бот: команда /start, базовая FSM-машина онбординга (вопросы по шагам)
  • ✅ OCR pipeline: получить фото → Tesseract → LLM-парсер → JSON → паспорт в БД
  • ✅ MCQ-fallback если OCR пропустил поля или нет фото у пользователя
  • ✅ "Готовый паспорт" showcase в TG
  • Acceptance: реальный пользователь (тестер из команды) проходит онбординг за < 5 минут с фото и < 8 без.

Week 3: PWA + read-only passport view

  • ✅ PWA скелет: React + Vite + service worker
  • ✅ Авторизация через TG-login или email-magic-link
  • ✅ Паспорт UI (просмотр + редактирование)
  • ✅ QR-share: одна кнопка → ссылка на read-only страницу
  • ✅ Read-only passport view (без авторизации)
  • Acceptance: пользователь может открыть PWA, видит свой паспорт, может поделиться через QR. Тестер ветеринар (любой, кого мы попросим) открывает QR-ссылку и видит чистый паспорт.

Week 4: Прививки + еженедельный чек-ин

  • ✅ Vaccination calendar logic (стандартный календарь по виду + породе + региону)
  • ✅ Cron: ежедневная проверка «есть ли прививка через 14/7/1 день?»
  • ✅ Push-уведомление в TG/PWA с кнопками
  • ✅ Weekly check-in: форма с 3-4 закрытыми вопросами + 1 открытым (текст + voice)
  • ✅ Voice capture → S3 (без транскрипции пока)
  • Acceptance: пользователь получает push с прививкой за 14 дней. Пользователь получает push в воскресенье вечером, может ответить за 30 секунд.

Week 5: Batch worker + summary

  • ✅ Cron worker: транскрипция голосовых через Whisper (CPU batch)
  • ✅ Cron worker: monthly summary через router LLM (Qwen 3.6 self-hosted)
  • ✅ Push доставка summary в TG/PWA
  • ✅ Internal admin-page для просмотра логов и проверки качества summary
  • ✅ End-to-end QA на 5 фейковых паспортах (с разными видами/возрастами/прививками)
  • Acceptance: через 4 еженедельных опроса пользователь получает осмысленный summary. Качество summary проверено на 5 фейковых животных.

Week 6: Buffer / polish / launch prep

  • ✅ Bug fixes из QA week 5
  • ✅ UX polish (Onboarding micro-copy, push-tone, error states)
  • ✅ Подготовка launch-материалов: 2-3 поста для TG-каналов (Юлия)
  • ✅ Smoke-test на 10 живых пользователях из ближайшего круга
  • ✅ Документация (README, deployment guide, basic ops doc)
  • Acceptance: ничего критичного в Sentry, smoke-test 10/10 пользователей завершили онбординг, paspport-share работает, прививочный пуш приходит в эмуляции.

5 · Acceptance Criteria — Phase-1 launch ready

Мы запускаемся в публичный пилот только когда выполнены все 7 пунктов:

  1. ✅ Онбординг конверсия на 10 тестерах ≥ 60% (создали паспорт от клика)
  2. ✅ Time-to-paspport ≤ 5 минут с фото / ≤ 8 минут без на средних тестерах
  3. ✅ QR-share работает в Safari + Chrome + любом vet-планшете (3 устройства проверены)
  4. ✅ Vaccination reminder приходит точно (тестер увидел push в эмуляции 14/7/1 день)
  5. ✅ Weekly check-in: тестер прошёл 4 опроса, получил summary, summary осмыслен (subjective check by Юля)
  6. ✅ Sentry: 0 critical errors за 48 часов smoke-test
  7. ✅ Variable cost после 10 тестеров × 4 недели — посчитан, в пределах $0.01/user/mo

Если что-то из этого не закрыто — не запускаем. Анкер бюджета не значит «ship at all costs».
